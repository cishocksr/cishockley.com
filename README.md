# cishockley.com

Personal portfolio and blog for Chris Shockley, a software engineer specializing in modern web development.

## Tech stack

- **Framework**: Next.js 16 (App Router), React 19
- **Styling**: Tailwind CSS v4
- **Content**: [Velite](https://velite.dev) — MDX in `content/` compiled to `.velite` at build time
- **UI**: Radix UI (Dialog, ScrollArea, etc.), Framer Motion, next-themes (dark/light)
- **Tooling**: Biome (lint + format), TypeScript

## Getting started

**Prerequisites**: Node.js, and a package manager (Bun, npm, pnpm, or yarn). This repo uses [Bun](https://bun.sh).

```bash
bun install
bun run dev
```

Open [http://localhost:3000](http://localhost:3000). The dev script runs Velite in watch mode and Next.js together, so edits to MDX files in `content/` are picked up automatically. On first run, Velite generates the `.velite` folder from `content/`.

**Build and run in production:**

```bash
bun run build
bun run start
```


## Project structure



- **`app/`** — Next.js App Router: `(main)/` (home, about, contact), `blog/`, `projects/` (including `[slug]` dynamic routes).
- **`content/`** — Source content for Velite:
  - **`content/blog/**/*.mdx`** — Blog posts (frontmatter: `title`, `slug`, `date`, `description`, `published`, `tags`).
  - **`content/projects/**/*.mdx`** — Projects (frontmatter: `title`, `slug`, `date`, `description`, `image`, `projectUrl`, `githubLink`, `tags`, `status`, `featured`).
- **`components/`** — Shared UI (header, footer, motion, skills, etc.).
- **`config/`** — Site metadata and nav ([config/site.ts](config/site.ts)), social links.
- **`.velite/`** — Generated by Velite. Schema is defined in [velite.config.ts](velite.config.ts).

## Adding content

- **New blog post**: Add `content/blog/your-slug.mdx` with frontmatter (`title`, `slug`, `date`, `description`, `published`, `tags`). Body is MDX. With `bun run dev` running, Velite will pick it up.
- **New project**: Add `content/projects/your-slug.mdx` with project frontmatter (including optional `image`, `projectUrl`, `githubLink`, `status`, `featured`). See [content/projects/vetconnect.mdx](content/projects/vetconnect.mdx) for an example.

## Environment variables

| Variable | Purpose |
| -------- | ------- |
| `NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY` | Required for the contact form. Get a key at [web3forms.com](https://web3forms.com). |

Copy `.env.example` to `.env.local` and fill in the value for local development or deployment.
